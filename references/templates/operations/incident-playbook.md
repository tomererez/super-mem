# Incident Playbook

> What to do when things go wrong.

## Severity Levels
| Level | Description | Response time |
|-------|-------------|---------------|
| P0 - Critical | System down, data loss | Immediate |
| P1 - High | Major feature broken | Within 1 hour |
| P2 - Medium | Minor feature broken | Within 1 day |

## Response Steps
1. **Assess:** What's broken? Who's affected?
2. **Communicate:** Notify stakeholders
3. **Fix or rollback:** Ship a fix or revert
4. **Post-mortem:** Document what happened and why
